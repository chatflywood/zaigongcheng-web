**做了什么：**

## 04 作品 · 数据工具

### 作品一：工程建设数据驾舱（zaigongcheng-web）

**背景：** 工程建设管理涉及大量Excel报表数据——在建工程进度、预算执行、转固情况等，数据分散在多个文件中，每次汇报都需要手动汇总、反复核对，耗时且容易出错。

**做了什么：**

独立设计开发了一套工程建设数据自动化分析平台，从前端到后端一人完成，经过28个版本迭代，核心功能包括：

| 模块 | 功能 |
|------|------|
| 在建工程看板 | 上传Excel自动生成KPI卡片、管理员支出排名、工程明细下钻 |
| 四类工程预警 | 自动识别A/B/C/D四类工程风险等级，60天预警窗口 |
| 关键指标页 | 杂志风双栏布局，四环图+AI分析+工作事项，支持投屏演示模式 |
| 预算立项分析 | 预算执行进度跟踪、批次下达管理（23个专业可配置） |
| 数据档案库 | 三类年度文档管理，浏览器内Excel/Word在线预览 |
| AI智能分析 | 接入MiniMax大模型，自动生成管理汇报版/执行跟踪版分析 |
| 消息推送 | 飞书/企业微信webhook一键推送数据摘要 |
| Excel导出 | 转固推进清单、预警报表等一键导出 |

**技术栈：**
- 前端：Vue 3 + Vite + ECharts + Electron
- 后端：FastAPI + SQLAlchemy + pandas + OpenPyXL
- AI：MiniMax API（规则信号计算 + 大模型结构化输出）
- 设计：暗色毛玻璃风格，SVG动态仪表盘，数字跳动动画

**成果：**
- 将原本每次汇报需要2-3小时的数据汇总工作缩短到**上传Excel后秒级出图**
- 实现从"人工翻表"到"一键看板"的工作方式转变
- 7周内完成28个版本迭代，从零到完整可用系统

**可复用的东西：** Excel解析+指标计算的数据处理pipeline、四类工程预警算法、AI结构化分析prompt模板、飞书/企微消息推送组件。

## 05 作品 · AI应用

### 作品一：AI辅助工程文档生成工作流

**背景：** 工程管理中有大量周期性文档需要编写——月度分析报告、审计自查材料、投资评估报告等，格式相对固定但数据每次都变，重复劳动多。

**做了什么：**
- 将文档写作拆解为"数据提取 → 信号计算 → AI生成 → 人工审核"四个步骤
- 设计结构化prompt模板，让AI输出符合政企文档风格的内容
- 支持两种分析风格切换：管理汇报版（面向领导）和执行跟踪版（面向团队）
- 规则信号先行计算（趋势、风险、异常），AI基于信号生成分析，避免幻觉

**成果：**
- 月度分析报告编写时间从**2-3小时缩短到30分钟**（含人工审核修改）
- 文档质量更稳定，关键数据和风险点不会遗漏

**可复用的东西：** 结构化prompt模板、信号计算规则、双风格切换机制。

---

### 作品二：Claude Code在数据分析中的应用

**背景：** 工程数据分析涉及Excel解析、指标计算、可视化呈现等多个环节，传统方式需要在Excel和PPT之间反复切换。

**做了什么：**
- 使用Claude Code作为编程助手，通过自然语言描述需求快速生成代码
- 在工程建设数据驾舱的开发过程中，AI辅助完成前端组件、后端API、数据处理逻辑
- 探索"vibe coding"工作模式——描述业务意图，让AI生成实现代码

**成果：**
- 7周内独立完成一个完整的全栈数据平台（28个版本迭代）
- 开发效率相比纯手写提升约**3-5倍**，尤其在前端样式调试和API对接环节

**可复用的东西：** vibe coding工作方法、AI辅助开发的最佳实践（何时信任AI输出、何时需要人工校验）。

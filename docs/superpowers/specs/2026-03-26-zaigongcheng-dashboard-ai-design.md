# 在建工程大屏 AI 分析与重点工作功能设计规格

## 1. 概述

### 1.1 项目背景

zaigongcheng-web 的 KeyIndicators.vue 目前是"关键指标大屏"，展示立项进度、资本性支出进度、综合转固率三个环形仪表盘。现需要升级为更适合**项目经理和领导**查看的展示大屏，新增：
1. **AI 智能分析** — 接入 MiniMax 大模型，自动生成在建工程执行情况的分析报告
2. **近期重点工作** — 手动维护的重点工作列表，责任人、完成日期一目了然

### 1.2 需求目标

将 KeyIndicators.vue 升级为"工程进度大屏"，在大屏上按以下顺序展示：
1. **三大仪表盘** — 现有立项进度、资本性支出进度、综合转固率（保持不变）
2. **AI 分析报告** — MiniMax 生成的结构化分析报告 + 综合报告
3. **近期重点工作** — 可编辑的重点工作卡片列表

### 1.3 数据来源

**AI 分析数据来源**：
- 当前最新的 `zaigong_metrics` 数据（累计支出、待收货、本月支出、转固率、进度等）
- 当前最新的预算指标（年度预算、立项进度、年度支出、已占用、预占用）
- 从 `summary` 数据中获取各管理员的支出排名、月度推进、待收货压力、结转额等信息
- 根据分析日期计算当前月份应有节奏、目标缺口、预算兑现差、风险等级等规则信号
- 调用 MiniMax API 生成分析文本

**重点工作数据来源**：
- 纯前端 `localStorage` 存储
- 字段：`id`、`content`（工作内容）、`owner`（责任人）、`dueDate`（完成日期，YYYY-MM-DD 格式）、`status`（pending/completed）、`createdAt`

### 2.2 AI 分析模块

#### 2.2.1 UI 样式

- **卡片样式**：圆角 28px，边框 `rgba(0, 212, 255, 0.15)`，背景渐变同现有仪表盘
- **标题栏**：左侧 "🤖 AI 工程进度分析"，右侧 "刷新分析" 按钮（青色边框按钮）
- **加载状态**：刷新时按钮显示 loading 动画，卡片内容显示"分析中..."骨架屏
- **分析维度**（按固定顺序输出）：
  1. **综合评估** — 一段话总结整体情况
  2. **进度评估** — 当前完成率 vs 序时进度，是否滞后
  3. **支出分析** — 各管理员支出对比，支出最高/最低点评
  4. **转固率分析** — 综合转固率是否达标
  5. **风险预警** — 待收货压力、滞后风险等
  6. **下月预测** — 按当前月均支出预测下月累计完成情况

#### 2.2.2 刷新逻辑

- **页面首次加载**：从 `localStorage` 读取缓存的 AI 分析结果并显示
- **点击"刷新分析"**：
  1. 按钮进入 loading 状态
  2. 调用后端 `/api/ai/analyze` 接口
  3. 拿到结果后存入 `localStorage`（key: `ai_analysis_cache_v2`）
  4. 更新卡片内容
  5. 按钮恢复
- **缓存结构**（localStorage）：
  ```json
  {
    "content": "**综合评估**\n当前累计资本性支出...",
    "structured": {
      "overall": "……",
      "actions": ["……"],
      "basis": ["……"]
    },
    "generatedAt": "2026-03-26T10:30:00Z",
    "mode": "management",
    "snapshotHash": "..."
  }
  ```
- **缓存失效**：如果当前在建工程指标、预算指标、管理员月度推进或分析风格发生变化，自动标记缓存过期，显示"数据已更新，请点击刷新"提示

#### 2.2.4 错误处理

| 场景 | 处理方式 |
|------|----------|
| API 调用失败 | 显示红色提示"AI 分析生成失败，请稍后重试"，保留上一次缓存内容 |
| MiniMax 返回格式异常 | 捕获异常，写入错误日志，显示友好提示 |

## 3. 后端接口设计

### 3.1 新增接口

#### POST `/api/ai/analyze`

调用 MiniMax API 生成工程进度分析报告。

**请求体**：
```json
{
  "metrics": {
    "year_target": 503.0,
    "total_current": 245.3,
    "progress_pct": 48.8,
    "month_spend": 85.0,
    "pending": 68.5,
    "transfer_rate": 0.623
  },
  "budget": {
    "total_budget": 1260.0,
    "approval_progress_pct": 61.5,
    "annual_spend_total": 245.3,
    "occupied_total": 510.0,
    "preoccupied_total": 92.0
  },
  "analysis_date": "2026-03-27",
  "summary": [
    {
      "manager": "伍建勋",
      "capital": 85.2,
      "pending": 15.0,
      "rate": 0.65,
      "month_spend": 20.0,
      "transfer": 8.5
    }
  ]
}
```

**MiniMax API 调用**：
- 模型：`abab6.5s-chat`（海螺 AI 主力模型，支持中文，响应快）
- API 地址：`https://api.minimax.chat/v1/text/chatcompletion_pro?GroupId=xxx`（需替换为实际地址）
- 环境变量：`MINIMAX_API_KEY`、`MINIMAX_GROUP_ID`

**响应结构**：
```json
{
  "success": true,
  "data": {
    "content": "**综合评估**\n当前累计资本性支出 245.30 万元，完成年度目标 48.8%，整体进度正常。\n**进度评估**\n...",
    "generated_at": "2026-03-26T10:30:00Z",
    "model": "abab6.5s-chat"
  }
}
```

**错误响应**：
```json
{
  "success": false,
  "error": "AI_SERVICE_ERROR",
  "message": "MiniMax API 调用失败，请稍后重试"
}
```

### 3.2 .env 配置

```
MINIMAX_API_KEY=your_api_key_here
MINIMAX_GROUP_ID=your_group_id_here
MINIMAX_API_URL=https://api.minimax.chat/v1/text/chatcompletion_pro
```


**新增依赖**：
- 前端 Markdown 渲染：`marked`（或用简单的正则替换，因为 AI 输出是固定格式，不需要完整 MD 解析）

### 4.2 API 接口调用

**新增 `frontend/src/api/index.js`** 中的调用：
```javascript
export const generateAIAnalysis = async (metrics, summary) => {
  const res = await axios.post('/api/ai/analyze', { metrics, summary })
  return res.data
}
```

## 6. 验收标准

### AI 分析模块
1. 页面加载时显示缓存的分析结果（秒开）
2. 点击"刷新分析"按钮后，调用后端接口，Loading 状态正常
3. AI 返回的分析内容按 6 个固定维度展示（综合评估、进度评估、支出分析、转固率分析、风险预警、下月预测）
4. 数据更新后（上传新 Excel），缓存自动失效，显示"数据已更新"提示
5. 接口失败时显示友好错误提示，不影响页面其他功能
6. API Key 仅存在于后端 `.env`，前端无法获取

### 重点工作模块
1. 可正常添加新工作（内容、责任人、完成日期必填）
2. 可编辑已有工作
3. 可删除工作（带确认提示）
4. 可标记完成/未完成
5. 刷新页面后数据保持（localStorage 正常）
6. 已完成的工作排在列表最后，且样式有区分
7. 列表按完成日期升序排列

### 整体
1. 新增模块风格与现有大屏风格一致（深色主题、青色/紫色/绿色点缀）
2. 页面在大屏/投影展示时布局正常，不溢出
3. 响应式布局在普通显示器上正常显示

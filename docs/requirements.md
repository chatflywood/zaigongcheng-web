# 工程数据分析平台需求文档

## 1. 文档目的

本文档用于统一记录”工程数据分析平台”的业务目标、页面功能、数据口径、模板要求和后续迭代方向，降低后续开发和需求沟通成本。

适用范围：
- 在建工程分析
- 预算立项分析
- 关键指标总览

**当前版本：v1.4（2026-04-07）**

---

## 2. 项目背景

当前工作场景中，需要频繁处理两类 Excel 数据：
- 在建工程明细总表
- 预算执行情况（预算占用）

业务侧希望系统能完成以下事情：
- 上传 Excel 后自动分析，不再手工整理
- 以页面方式快速查看核心指标、专业分布和项目明细
- 支持日常跟踪、周通报、月度汇报
- 支持基于历史上传数据做简单对比

---

## 3. 核心目标

### 3.1 业务目标

- 快速掌握当期资本性支出完成情况
- 快速掌握预算下达后的占用和立项推进情况
- 用统一页面汇总关键指标，便于管理层查看
- 降低手工汇总和口径不一致的问题

### 3.2 系统目标

- Excel 上传后可直接得到结构化分析结果
- 关键口径尽量固定，避免页面和表格口径漂移
- 页面布局适合业务人员日常查看和截图汇报
- 支持后续继续扩展导出、筛选、预警等能力

---

## 4. 用户与使用场景

### 4.1 主要用户

- 分公司建设相关管理人员
- 工程分析和预算管理人员
- 需要查看阶段性进度的领导或汇报人员

### 4.2 典型使用场景

- 每日或每周上传最新在建工程明细，查看资本性支出进度
- 上传最新预算执行情况，查看预算占用和新建项目分布
- 在关键指标页查看三类核心指标的综合表现
- 截图或口头汇报时快速定位“合计、差额、进度、异常点”

## 5. 页面范围与功能

## 5.1 顶部区域

### 功能目标

- 提供系统导航和当前数据状态概览

### 当前展示内容

- 页面导航：
  - 在建工程
  - 预算立项
  - 关键指标
  - 历史记录
- 状态模块：
  - 当前视图
  - 数据准备度
  - 在建工程
  - 预算立项

## 5.2 在建工程页面

### 页面目标

- 展示资本性支出完成进度、待收货压力、管理员排名和明细情况

### 上传区

- 左侧说明卡：
  - 说明上传用途
  - 提供“设置当期资本性支出目标”输入框
- 右侧上传卡：
  - 支持拖拽上传
  - 支持点击上传
  - 上传卡结构和预算立项页保持一致

### 上传规则

- 目标金额必须由用户手动输入
- 未输入目标金额时，不允许上传
- 上传成功后页面进入数据展示模式

### 数据展示区

- 顶部指标卡：
  - 本年累计资本性支出
  - 已下单待收货
  - 本月资本性支出
  - 综合转固率
- 预警条：
  - 待收货压力
  - 转固率状态
  - 目标差额
- 主区域：
  - 左侧资本性支出进度仪表盘
  - 右侧工程管理员支出排名图
- 四类工程预警面板：
  - 四张统计卡片：列账不及时、预转固不及时、关闭不及时、长期挂账
  - 每张显示「已触发数 / 预警数」
  - 点击卡片展示该类型明细弹窗
  - 点击模块标题展示全部四类预警明细（分组显示）
  - 导出预警清单按钮（位于明细弹窗右上角）
  - 预警窗口固定为 60 天
  - 数据过滤：排除「局房及基础设施」和「工程已关闭」的记录
- 四类工程预警明细弹窗：
  - 标题显示"四类工程预警明细"或具体类型名称
  - 数据日期显示在标题后面（字体较小）
  - 明细表格列：状态、工程名称、验收类型、管理员、关键日期、截止日期、工程状态、天数、处置建议
  - 不显示编号、一级专业列
  - 处置建议列支持换行显示
- 表格：
  - 各工程管理员汇总（点击「当期资本性支出进度」行展开/收起）
  - 支持查看较上期变化
  - 支持点击管理员查看工程明细弹窗

### 历史对比规则

- 上传结果保存到数据库
- 相同文件名上传时覆盖旧记录
- 页面内只负责查看历史快照，不直接展示历史对比结果
- 双版本历史对比统一在顶部“历史记录”入口中完成
- 对比时由用户手动选择两条记录，分别作为 `A` / `B`
- 对比结果必须明确标注两条记录对应的日期
- 在建工程对比范围收敛为：
  - 资本性支出进度
  - 转固率
  - 管理员推进 Top 5

## 5.4 关键指标页面

### 页面目标

- 汇总展示资本性支出进度、立项进度和综合转固率

### 当前展示内容

- 立项进度仪表盘
- 资本性支出进度仪表盘
- 综合转固率仪表盘
- AI 工程进度分析（配置检测、手动刷新、结果缓存）
- AI 分析风格切换（管理汇报版 / 执行推进版）
- AI 结构化结果展示（综合评估、进度评估、支出分析、转固率分析、风险预警、下月预测、重点动作、分析依据）
- 近期重点工作（新增、编辑、删除、完成状态切换）
- 聚焦布局：3 指标 + AI + 重点工作

### 关键规则

- 资本性支出进度中的“目标”必须显示真实上传目标值
- 若没有有效目标值，显示 `—`
- 不允许回退显示固定默认值 `503`
- AI 未配置时需要明确提示，不可误导为分析完成
- AI 接口应支持 `style` 参数，并优先返回可直接渲染的结构化字段；新版分析需支持预算联动输入与分析依据输出

## 7. 数据口径与计算规则

## 7.1 在建工程

### 本年累计资本性支出

- 来源：在建工程明细中的 `本年累计资本性支出`

### 已下单待收货

- 公式：`累计订单金额(列帐及时性) - 累计收货金额(列帐及时性)`

### 综合转固率

- 公式：

```text
转固率 = 1 - (期末在建工程余额 + 期末工程物资) / (本年累计资本性支出 + 期初在建工程余额 + 期初工程物资)
```

### 四类工程预警

#### 数据过滤

- 排除条件：
  - 一级专业 = "局房及基础设施"
  - 工程关闭状态 = "工程已关闭"

#### 可配置参数

| 参数 | 值 | 说明 |
|---|---|---|
| 预警窗口 | 60 天 | 距截止日期不足 60 天纳入预警 |
| 预转固截止天数 | 60 天 | 初验批复后 60 天内须完成预转固 |
| 一次验收关闭截止天数 | 150 天 | 一次验收终验批复后 150 天内须正式转固 |
| 两次验收关闭截止天数 | 90 天 | 两次验收终验批复后 90 天内须正式转固 |

#### 派生字段

- **实际预转固日期**：优先取 `预转固日期`，无则取决算转固日期
- **收货率**：`累计收货金额(列帐及时性) / 累计订单金额(列帐及时性)`（当订单金额 > 0 时）

#### TYPE A — 列账不及时

触发前提：初验批复日期不为空

- 已触发：收货率 < 85%
- 预警：85% ≤ 收货率 < 90%

关键日期 = 初验批复日期，截止日期 = 无，天数文字 = "收货XX%"

#### TYPE B — 预转固不及时

触发前提：初验批复日期不为空

- 截止日期 = 初验批复日期 + 60 天
- 已触发（未完成）：实际预转固日期为空 且 截止日期 < 今日
- 已触发（超期完成）：实际预转固日期不为空 且 实际预转固日期 > 截止日期
- 预警：实际预转固日期为空 且 距截止日期 ≤ 60 天

#### TYPE C — 关闭不及时

触发前提：终验批复日期不为空（有效池中均未关闭）

- 截止日期优先取系统字段「应关闭日期」；若为空则自算：
  - 一次验收：终验批复日期 + 150 天
  - 两次验收：终验批复日期 + 90 天
- 已触发：截止日期 < 今日
- 预警：距截止日期 ≤ 60 天

#### TYPE D — 长期挂账

触发前提：长期挂账建议关闭日期不为空（系统已预算，直接使用）

- 已触发：长期挂账建议关闭日期 < 今日
- 预警：距长期挂账建议关闭日期 ≤ 60 天

## 7.2 预算立项

### 年度预算

- 来源：预算汇总 sheet 中各专业对应的年度预算值

### 年度支出

- 来源：最近一次在建工程上传数据
- 取值方式：按一级专业汇总 `本年累计资本性支出`
- 当前实现为：
  - 优先使用最近一次在建工程记录中可用的一级专业数据
  - 若数据库中缺少一级专业，则回查本地对应的在建工程源文件

### 已占用

- 来源：项目明细 sheet
- 取值方式：按一级专业汇总项目 `占用规模`

### 预占用

- 来源：项目明细 sheet
- 取值方式：按一级专业汇总项目 `预占规模`

### 小计

- 公式：`已占用 + 预占用`

### 立项进度

- 公式：`小计 / 年度预算`

### 支出进度

- 公式：`年度支出 / 年度预算`

## 8. 数据存储与历史规则

### 当前数据库

- SQLite
- 路径：`backend/data/analysis.db`

### 当前表

- `zaigong_records`
- `budget_records`

### 当前保存策略

- 在建工程上传历史已保存
- 预算上传历史已保存，并支持历史快照恢复

### 当前限制

- 在建工程记录中已保存管理员维度明细，但尚未稳定保存“一级专业”
- 因此预算页“年度支出”仍可能依赖本地源文件回查

---

## 9. 非功能要求

### 性能

- 常规 Excel 上传后应在可接受时间内返回分析结果
- 页面适合办公环境下桌面浏览器使用

### 易用性

- 页面需要适合截图汇报
- 页面布局优先“清楚、稳定、可快速扫读”
- 关键总量优先放在上方

### 可维护性

- 模板变更后应尽量通过后端解析兼容处理
- 文档、页面、数据口径三者应保持同步

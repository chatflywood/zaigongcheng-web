# 前端说明

前端基于 Vue 3 + Vite，负责在建工程分析、预算立项分析、关键指标总览以及全局历史记录中心的交互与可视化展示。

## 启动

```bash
cd /Users/liangsha/Documents/zaigongcheng-web/frontend
npm install
npm run dev
```

默认开发地址为 [http://localhost:5173](http://localhost:5173)。

## 构建

```bash
cd /Users/liangsha/Documents/zaigongcheng-web/frontend
npm run build
```

## 页面结构

- `src/App.vue`
  负责系统导航、顶部状态模块、全局历史记录中心和三个主页面切换。
- `src/views/Dashboard.vue`
  在建工程分析页，包含目标输入、文件上传、指标卡、进度仪表盘、四类工程预警面板（统计卡片 + 分类明细弹窗 + 导出）、管理员排名和明细弹窗。
- `src/views/Budget.vue`
  预算立项页，包含预算文件上传、指标卡、专业进度表和项目明细表。
- `src/views/KeyIndicators.vue`
  关键指标页，聚焦展示资本性支出进度、立项进度和综合转固率，并提供 AI 分析与重点工作维护。
- `src/api/index.js`
  前端接口封装，统一调用后端分析 API。

## 最近前端优化

- 顶部区域改为更紧凑的“标题条 + 4 个状态模块”结构。
- 在建工程页和预算立项页的上传区统一为双栏布局。
- 在建工程上传区改为与预算立项一致的上传卡结构和交互表现。
- 在建工程目标值改为用户手动输入，不再展示默认目标提示。
- 关键指标页的资本性支出目标值改为显示真实上传目标，不再固定回退到 `503`。
- 关键指标页新增 AI 工程进度分析区块（状态检查、手动刷新、缓存与失效策略）。
- AI 分析区新增风格切换（管理汇报版 / 执行推进版）。
- AI 结果渲染升级为结构化区块 + 重点动作列表。
- AI 分析已升级为预算联动模式，前端会同时提交在建工程指标、预算指标、管理员月度推进明细和分析日期。
- AI 结果新增“分析依据”展示区，用于解释结论引用的关键数字。
- AI 本地缓存升级为 `ai_analysis_cache_v2`，避免页面继续显示旧版分析结果。
- 关键指标页新增近期重点工作区块（新增、编辑、删除、状态切换，localStorage 持久化）。
- 关键指标页已隐藏非核心汇总模块，页面聚焦”3 指标 + AI + 重点工作”。
- 顶部新增全局历史记录中心，支持统一查看历史版本与双版本对比。
- 在建工程页新增四类工程预警面板：四张统计卡片横排展示（列账不及时/预转固不及时/关闭不及时/长期挂账），点击卡片查看分类明细，点击标题查看全部分组明细，预警天数 ≤ 30 天的行显示 ⚠️ 标记，支持导出预警清单。
- 「当期资本性支出进度」行支持点击展开/收起「各工程管理员汇总」表格。
- 历史对比统一在历史记录中心完成，业务页面仅保留历史快照查看。
- `VITE_API_BASE` 可用于覆盖默认 API 地址，默认仍为 `http://localhost:8000/api`。

# 工程数据分析平台

在建工程和预算下达的自动化分析系统，支持数据上传、可视化展示和关键指标总览。

---

## 项目概述

本项目用于分析在建工程数据和预算下达立项进度数据，自动计算关键指标并生成可视化大屏展示。

## 功能说明

### 1. 在建工程分析

**上传文件要求：**
- 格式：Excel (.xlsx, .xls)
- 必须包含列：工程管理员、工程名称、结转额、本年累计资本性支出、累计订单金额、累计收货金额、本月资本性支出、在建工程期末余额、工程物资、在建工程年初数、工程物资年初数

**核心指标：**
- 本年累计资本性支出
- 已下单待收货
- 本月资本性支出
- 综合转固率

**转固率计算公式：**
```
转固率 = 1 - (期末在建工程余额 + 期末工程物资) / (本年累计资本性支出 + 期初在建工程余额 + 期初工程物资)
```

### 2. 预算下达及立项进度分析

**上传文件要求：**
- 格式：Excel (.xlsx, .xls)
- 必须包含 sheet：预算下达及立项进度
- 项目明细 sheet 支持自动识别以“新建项目明细”结尾的名称，如 `2026年新建项目明细`、`2027年新建项目明细`

**核心指标：**
- 年度预算
- 已占用
- 预占用
- 立项进度

### 3. 关键指标进度

显示三大核心指标的汇总展示：
- 资本性支出进度
- 立项进度
- 综合转固率

## 页面说明

### 导航栏

顶部区域包含 4 个状态模块：
- **当前视图** - 当前所在页面
- **数据准备度** - 两类分析数据的完成情况
- **在建工程** - 在建工程数据最近上传日期
- **预算立项** - 预算数据最近上传日期

说明：
- 以上 4 个状态模块仅在在建工程/预算立项视图显示
- 在关键指标视图下会自动隐藏，避免干扰核心指标阅读

导航栏仍包含三个页面入口：
- **在建工程** - 在建工程数据分析
- **预算立项** - 预算下达及立项进度
- **关键指标** - 三大核心指标总览（需上传两个分析数据后才可访问）
- **历史记录** - 统一查看在建工程和预算立项历史版本，并在历史中心内完成双版本对比

已上传数据的页面标签会显示蓝色指示点。

### 在建工程分析页

**上传模式：**
- 左侧说明卡用于录入当期资本性支出目标
- 目标金额必须由用户手动输入后才可上传
- 右侧上传卡支持拖拽或点击上传 Excel 文件
- 上传卡结构和预算立项页保持一致

**数据展示模式：**
- 顶部 4 个指标卡片
- 左侧：当期资本性支出进度仪表盘
- 右侧：工程管理员支出排名柱状图（显示较上期变化）
- 下方：各工程管理员汇总表格（含较上期变化列）

**历史查看功能：**
- 支持从页面内历史记录面板查看某次上传快照
- 支持从全局历史记录中心查看某次上传快照
- 相同文件名上传时自动覆盖

### 预算立项页

**上传模式：**
- 左侧说明卡展示文件建议和关键产出
- 右侧上传卡支持拖拽或点击上传预算 Excel 文件

**数据展示模式：**
- 顶部 5 个指标卡片
- 各专业预算立项进度表格
- 2026年新建项目明细表格

**历史查看功能：**
- 支持从页面内历史记录面板查看某次上传快照
- 支持从全局历史记录中心查看某次上传快照

### 历史记录中心

- 顶部统一入口进入
- 支持切换查看“全部 / 在建工程 / 预算立项”
- 支持为同一模块选择两条历史记录进行对比
- 对比结果统一展示在历史记录中心，不在业务页面重复展示
- 对比区会明确显示两条记录对应的日期
- 当前支持的精简对比项：
  - 在建工程：资本性支出进度、转固率、管理员推进 Top 5
  - 预算立项：年度预算、立项进度、专业推进 Top 5

### 关键指标页

- 三个核心进度卡片（立项进度、资本性支出进度、综合转固率）
- 四色顶部描边（紫/青/蓝/红）+ 玻璃拟态卡片风格（暗色背景 #080C12 + backdrop-filter blur）
- 仪表盘升级为 SVG 弧线可视化（stroke-dashoffset 绑定数据 + 描边动画）
- 支持全屏展示模式：进入后铺满整个屏幕，去掉四周留白
- 大屏专用导航栏：品牌 Logo + 实时状态（绿色脉冲点）+ 进入/退出展示模式按钮
- 新增 AI 工程进度分析区块（状态检测、手动刷新、缓存与失效提示）
- AI 分析可视化重构：状态行（三色状态点 + 一句话摘要）+ 三格指标（数字优先）+ 重点动作（优先级 tag + 责任人 chip）
- AI 内容后处理：自动将”差距 0.00 万元”等不合理表述替换为”已达标”
- 支持 AI 分析风格切换：管理汇报版 / 执行推进版
- AI 分析已升级为预算联动模式，会综合在建工程指标、预算进度、管理员月度推进与风险信号生成结论
- 新增近期重点工作区块（新增、编辑、删除、状态切换，localStorage 持久化）
- 资本性支出进度卡片中的目标值读取真实上传目标，不再固定显示默认值
- 页面布局聚焦为”3 指标 + AI 分析 + 重点工作”，移除非核心汇总条

## API 接口

### 在建工程分析

| 接口 | 方法 | 说明 |
|---|---|---|
| `/api/zaigong/upload` | POST | 上传 Excel，返回分析结果 |
| `/api/zaigong/metrics` | GET | 获取配置信息 |
| `/api/zaigong/history` | GET | 获取历史记录列表 |
| `/api/zaigong/history/{id}` | GET | 获取指定历史快照 |
| `/api/zaigong/compare` | GET | 获取最新数据与历史数据对比 |

### 预算分析

| 接口 | 方法 | 说明 |
|---|---|---|
| `/api/budget/upload` | POST | 上传预算 Excel，返回分析结果 |
| `/api/budget/history` | GET | 获取预算历史记录列表 |
| `/api/budget/history/{id}` | GET | 获取指定预算历史快照 |

### AI 分析

| 接口 | 方法 | 说明 |
|---|---|---|
| `/api/ai/status` | GET | 获取 AI 服务配置状态 |
| `/api/ai/analyze` | POST | 生成工程进度 AI 分析报告（支持 `style`，返回 `structured`） |

## 数据流向

```
Excel 文件
    ↓
后端 API (FastAPI)
    ↓
数据处理 (pandas)
    ↓
计算分析 (转固率、汇总、排名)
    ↓
SQLite 数据库存储
    ↓
返回 JSON
    ↓
前端展示 (Vue + ECharts)
```

**数据持久化：**
- 上传的文件数据自动存储到 SQLite 数据库
- 存储路径：`backend/data/analysis.db`
- 按文件名日期自动识别和对比数据

## 后续开发指南

### 添加新功能流程

1. **确定功能需求** - 明确要实现什么
2. **后端开发** - 在 `services/` 添加业务逻辑，在 `routers/` 添加 API
3. **前端开发** - 在 `views/` 添加页面组件，在 `App.vue` 添加导航
4. **测试** - 上传数据验证功能

### 修改现有功能

- **修改计算逻辑** → `backend/services/analysis.py`
- **修改页面样式** → `frontend/src/views/*.vue` 的 `<style>` 部分
- **修改页面结构** → `frontend/src/views/*.vue` 的 `<template>` 部分

## 常见问题

**Q: 上传文件失败？**
A: 检查 Excel 文件格式是否符合要求，确保包含必要的列和 sheet。

**Q: 数据不显示？**
A: 检查后端服务是否正常运行，确认端口 8000 未被占用。

**Q: 页面样式异常？**
A: 清除浏览器缓存，或尝试刷新页面。
